# Intent V2 Roadmap

## Goal

把题库 Agent 从“能完成单轮和预设多轮流程”升级为“能可靠理解连续指代，并且只能在安全动作空间中决策”。本阶段的目标是连续对话可靠性；LangGraph、LangChain 和自由工具调用都不是默认交付物。

## Scope

- 保留现有检索、章节判断、主库/字母库路由、粗筛、视觉复筛、工具层、FastAPI 和 SQLite 会话实现。
- 每个用户回合最多判断并执行一个高层动作；组合表达通过动作参数承载，例如选择第二题并覆盖章节。
- 规则处理明确表达；Qwen 只处理需要上下文理解的表达；代码最终校验状态、权限和参数。
- 只开放只读搜题动作。入库、删除、修复仍走现有 `plan -> confirm -> execute`，不进入 Intent V2 动作空间。
- 本阶段不迁移 LangGraph，不连接新的飞书 Agent，不改造现有题库飞书机器人。

## Delivery Steps

1. **冻结 V1 基线**：确认工作树，运行现有 Intent、Agent、会话恢复和全量测试，记录 V1 的输入、动作结果及已知失败类型。
2. **定义 ActionDecision V2**：确定动作集合、字段、题目/候选编号命名空间、章节覆盖、置信度和 `needs_clarification` 语义。
3. **建立权限矩阵**：逐个 phase 规定允许动作、必需参数、编号边界、追问条件和禁止动作；校验结果以代码为准，不信任模型自报置信度。
4. **建立评测集与评测器**：整理约 40 组真实多轮对话，覆盖连续选题、另一个/刚才那个/剩下那题、候选与题号冲突、章节修改、失败重试和组合表达，并跑出 V1 基线。
5. **实现 ConversationContext V2**：提供当前阶段、当前页面语义、题目及处理进度、选中题目、章节、候选摘要、最近用户/系统动作和当前允许动作；移除敏感或无关字段。
6. **实现 Intent V2 推理链**：保留规则快速通道，增加 Qwen 结构化动作、严格解析、代码校验、歧义追问和规则降级；模糊回合最多调用一次意图模型。
7. **接入 Agent 与 SQLite**：保持旧会话可加载，持久化新增上下文字段，保证重启恢复、章节纠正、多题继续、候选切换和答案回看不回退。
8. **对照验收与发布**：运行单元、多轮、API、持久化和全量回归；生成 V1/V2 对照结果，同步 README、SKILL 和项目记忆，只提交本阶段相关文件并推送。

## Acceptance Gates

- 禁止动作实际执行数为 0，越界题号和候选编号执行数为 0。
- 明确指令不低于 V1；指代类样本“正确执行或正确追问”达到 90% 以上。
- 每个模糊用户回合最多一次意图模型调用，规则可确定的表达不调用模型。
- 当前单题、多题、章节纠正、候选切换、失败重试、答案回看、SQLite 恢复和 FastAPI 测试全部通过。
- Intent V2 未证明提升前不替换稳定入口；先离线对照，必要时使用影子判断。

## LangGraph Decision Gate

只有动作协议和状态字段稳定、评测证明 Intent V2 有效，并且确实出现跨请求暂停/恢复、人工确认或现有状态机难以维护的分支时，才评估薄层 LangGraph。即使接入，也只编排成熟节点，不重写检索算法，不让模型自由决定写操作。
