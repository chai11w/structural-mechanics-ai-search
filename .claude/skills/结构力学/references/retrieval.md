# 检索流程

当用户想要根据图片或荷载描述查找相似题目时，使用此流程。

## Step 1: 确定输入

用户可能提供:
- **图片路径**: 一张结构力学题目图
- **荷载描述**: 文字描述如 "集中力10kN + 均布荷载q=4kN/m"
- **图片在对话中**: 用户粘贴了图片

## Step 2: 提取荷载

### 如果有图片
调用 search.py 识别荷载:
```bash
python "D:\桌面\答疑、帮做\结构力学\帮做\search.py" search --image "<图片绝对路径>" --chapter "<章节>"
```

### 如果只有文字描述
从描述中构造荷载 JSON，然后调用:
```bash
python "D:\桌面\答疑、帮做\结构力学\帮做\search.py" search --loads '{"loads":[{"type":"集中","raw":"10kN"},{"type":"均布","raw":"q=4kN/m"}]}' --chapter "<章节>"
```

从文字描述构造 JSON 的规则:
- 集中力: `{"type": "集中", "raw": "原始标注"}`
- 均布荷载: `{"type": "均布", "raw": "原始标注"}`
- 弯矩: `{"type": "弯矩", "raw": "原始标注"}`
- 按 集中 → 均布 → 弯矩 排序

## Step 3: 确定章节

用户可能说:
- "第2章" → `2静定结构`
- "静定结构位移" → `3静定结构位移`
- "力法" → `4力法`
- "位移法" → `5位移法`
- "力矩分配" → `6力矩分配`

如果不确定章节，询问用户。

## Step 4: 返回结果

脚本输出格式:
```
1. 相似度: 92%
   题目: 2静定结构/1单跨梁/题目a/1力/13.jpg
   路径: file:///D:/桌面/答疑、帮做/结构力学/帮做/2静定结构/1单跨梁/题目a/1力/13.jpg

2. 相似度: 75%
   题目: ...
```

默认返回 Top 5，按相似度降序。如果未识别到荷载，提示用户并列出章节内前几题。

## 相似度算法

- 荷载按类型分组 (集中/均布/弯矩)
- 组内用 Jaccard 相似度 (交集/并集)
- 双方都空的类型跳过不计
- 最终分 = 有效类型 Jaccard 均值
