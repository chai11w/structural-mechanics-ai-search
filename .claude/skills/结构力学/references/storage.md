# 储存流程

当用户想要将新题目存入 Excel 题库时，使用此流程。

## Step 1: 确定输入

用户可能提供:
- **图片路径**: 一张新题目图，需要自动识别荷载
- **路径+描述**: 已知图片路径和荷载内容

## Step 2: 储存

### 方式 A: 给图片 (自动识别)
```bash
python "D:\桌面\答疑、帮做\结构力学\帮做\search.py" store --image "<图片绝对路径>" --chapter "<章节>"
```
脚本会:
1. 调用 GLM-5V-Turbo 识别荷载
2. 自动生成相对路径 (相对于帮做目录)
3. 追加写入章节 Excel
4. 自动去重 (同路径覆盖)

### 方式 B: 给路径+荷载 (跳过识别)
```bash
python "D:\桌面\答疑、帮做\结构力学\帮做\search.py" store --path "<相对路径>" --loads '<荷载JSON>' --chapter "<章节>"
```
适用于用户已经知道荷载内容，或手动确认后写入。

相对路径格式示例: `2静定结构/1单跨梁/题目a/新题/1.jpg`

## Step 3: 确认

脚本输出写入结果，包括:
- 目标 Excel 文件名
- 当前总行数
- 新增行数

## 荷载 JSON 格式

```json
{"loads": [{"type": "集中", "raw": "10kN"}, {"type": "均布", "raw": "q=4kN/m"}, {"type": "弯矩", "raw": "10kN·m"}]}
```

规则:
- type 只能是 "集中"、"均布"、"弯矩"
- raw 保留图中原始标注
- 按 集中 → 均布 → 弯矩 排序
- 无荷载: `{"loads": []}`
